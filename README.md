# 📧 Email Spam Detector — Reinforcement Learning (Contextual Bandit)

A spam classifier that uses a **Contextual Bandit** (a reinforcement-learning formulation) to label emails/SMS as **spam** or **ham**, wrapped in an interactive **Streamlit** dashboard with rich data and model statistics.

## Why Contextual Bandit?

Standard supervised models learn from a fixed labeled dataset. A **contextual bandit** is the natural RL formulation for classification:

- **State (context)** = TF-IDF feature vector of the email
- **Action** = `0` (ham) or `1` (spam)
- **Reward** = `+1` for correct classification, `-1` for wrong
- **Policy** = linear function approximation: `score(a) = w_a · x`
- **Exploration** = ε-greedy with decay (start 0.3 → 0.05)

This lets the agent *learn online from feedback*, exactly the spirit of RL.

## Quick start

```bash
# 1. install
pip install -r requirements.txt

# 2. build features (first time only)
python preprocess.py

# 3. train the bandit
python train_model.py

# 4. launch dashboard
streamlit run app.py
```

The dashboard opens at `http://localhost:8501`.

## Dashboard features

### 🔮 Predict
- Paste any email/SMS
- Get **spam/ham** label with confidence bar
- See top tokens pushing toward spam vs. ham (interpretability)

### 📊 Data Stats
- Class distribution pie chart
- Message-length & word-count histograms
- Top-20 most frequent tokens for spam vs. ham
- Random sample viewer

### 🤖 Model Stats
- Accuracy / Precision / Recall / F1
- Confusion matrix heatmap
- Reward-per-episode learning curve
- Epsilon-decay plot
- Top learned weights per action

## Hyperparameters (in `train_model.py`)

| param        | default | meaning                                    |
|--------------|---------|--------------------------------------------|
| `alpha`      | 0.5     | learning rate                              |
| `epsilon`    | 0.3     | initial exploration probability            |
| `epsilon_min`| 0.05    | floor for ε after decay                    |
| `epsilon_decay` | 0.95 | multiplicative decay each episode        |
| `episodes`   | 15      | number of passes over the training set     |

## Notes

- The first time `preprocess.py` runs, NLTK downloads `stopwords` automatically.
- Encoding for `spam.csv` is `latin-1`.
- All long-lived outputs go into `artifacts/` — the dashboard loads them with `st.cache_resource`, so re-running training is just a refresh of the page.
